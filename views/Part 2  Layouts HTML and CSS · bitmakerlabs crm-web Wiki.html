This part of the tutorial assumes that you've completed Part 1.

This assignment will focus more specifically on how to best organize the HTML, CSS and views that we're using in our Sinatra application. Next off, you can choose to style it whichever way you like.

1. Using Layouts

Our application should currently hold at least 3 different views. Have you noticed anything about these views and the responses we've created so far?

They're not complete HTML documents, they're not even inside an tag!

If you decided to add them, that's great! But you probably ended up duplicating a lot of code, since each one of these views likely has very similar HTML overall.

As we already know, programmers like things DRY (Don't Repeat Yourself) and layouts luckily come to the rescue!

Layouts are a common part of any templating language, and this is the case with Sinatra. We'll become acquainted with these again when using Rails.

Here's what an example layout looks like:

<!DOCTYPE html>
<html>
  <head>

  </head>
  <body>
    <%= yield %>
  </body>
</html>

Layouts use yield, which you may have never seen before. yield allows us to dynamically insert a block of code inside this file, which means we can automatically wrap each one of our views with this full HTML document.

Create a full HTML document

Creating a layout in Sinatra is pretty straightforward: there's a convention for everything! If you create your layout as views/layout.erb, it will automatically get applied.

Build a layout that works for your web application in a new file called views/layout.erb. It should contain a full html document, including:

  • a DOCTYPE declaration
  • a <head> with a <title>
  • a <body> that yields to the views.

When you're done, commit your changes. Try viewing the source from all your different pages so far, you'll see that they're all using the layout.

2. Add a header, footer, and navigation

Typically layouts are pushed even further than just the bare minimum valid HTML, as they allow you to apply an overall look and feel to your entire application. For each page, give your application a consistent look by creating a page header, footer and a navigation in your layout.

Some of the following HTML 5 "section" elements might be good to consider adding to your layout: https://developer.mozilla.org/en/docs/Web/Guide/HTML/HTML5/HTML5_element_list#Sections

Once you're done, commit your changes and move onto styling and positioning.

3. Make It All Look Great!

Now that you've properly organized things, you'll need to style your application, whichever way you like, but here are a few guidelines:

  • Prefer external stylesheets over inline styles: you can store any stylesheets or images or other static files inside a "public" directory in your project. When you're linking them, don't forget to put a "/" in front of the file name so no matter where you can in project you can still find it.

Example, linking a stylesheet inside your layout's <head> tag.

<html>
  <head>
    <link rel="stylesheet" type="text/css" href="/style.css">

...
  • Try focusing on a two- or three- column layout: you can add multiple divs inside your <body tag and move the yield block to where the main content should belong.

Most of these styles can all be set up inside your new layout file, since it can be applied to all pages.

Happy Styling!
